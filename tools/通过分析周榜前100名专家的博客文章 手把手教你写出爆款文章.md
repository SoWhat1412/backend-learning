# 1. 文章布局
文章布局至关重要，爆文布局一般都有如下几个特性。
1. 标题好
> 都说酒香不怕巷子深，问题是现在都2020年了烙铁！快餐时代你文章标题都不好谁看你文章，好的文章一般避免使用冗长的标题，字数在30个汉字以内为最佳。
2. 目录清晰
> 多看下大佬们的文章，你会感觉到大佬的文章一般都带目录的，有目录的文章架构清晰，有条理，可随意跳转，方便自己也方便读者查看。
3. 摘要简明扼要
> 俗称引子，程序员面试的时候还要带简历做个简单介绍呢，你写这篇文章尽量要几句话说明原有。摘要一般在文章开头，我们可以理解为一篇文章的自我介绍，可以自己设置摘要，如果没有自己设置，系统默认抓取文章开头前200字在推荐页展示，咨询过官方人员建议后是建议自行设置展示文字。
4. 图片引人
>`一图胜千言`，再简单的问题尽量都图形化给读者展示出来，详细的`思维导图`总结与有趣的`表情包`等图片可以带来好的阅读体验增加点赞关注，
>PS：`图片尽量自己绘制且勿直接截图`！
>demo：[去银行办理业务还被业务员教了下线程池](https://sowhat.blog.csdn.net/article/details/105189304)  跟 [还看不懂JDK7 HashMap环的产生原理你来打我](https://sowhat.blog.csdn.net/article/details/105189304)

5. 标签
> 自己书写的文章在发布的最后一步尽量添加好标签哦，这样方便官方后台近精准推送。
> 
![在这里插入图片描述](https://img-blog.csdnimg.cn/20200418150606577.png#pic_center)
6. 一个文章尽量一个简单知识点
>  文章篇幅切勿太长哦，比如 [你连几个简单的HashMap都答不上来着实下饭啊](https://sowhat.blog.csdn.net/article/details/105000051) 这篇文干货满满不过太长了，没几个人愿意看下去的！

这几点无论你认可与否都是决定你文章质量的关键因素。
# 2. 类型建议
不同文章一般写作的套路是不一样的哦，简单列举两个技术类文章固定的套路。
1.解决问题的文章
>这种文章结构一般如下：
>> 1. 问题是怎么发现的，先复现。
>> 2. 排查问题的详细过程，如何追溯。
>> 3. 最终解决方案。
>> 3. 总结反思： 是否可以更快发现问题？其他解决思路等。

2. 技术点文章
> 书写一个知识点类型的文章套路一般是
> > 1. 是什么？简单介绍
>> 2. 为什么这么应用？
>> 3. 它的优缺点？
>> 4. 代码案例
>> 5. 它的原理?
>> 6. 同类型的对比。
>> 7. 源码阅读。
# 3. 干货建议
